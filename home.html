Project Libgen




Part 1: Usability


Introduction

The perception of a good UI has revolutionized in the past decade, and some design ideas in the past are no longer favored by younger users today. The "file-sharing based shadow library website" Library Genesis , though very useful for many students and scholars in getting access to books and articles, has a UI that could use some improvements to enhance user experience.

Usability Analysis




Learnability

The current UI is relatively easy to learn for a specific group of users. The frequent users of the website are generally students, scholars, or book lovers who wants to get access to free public recourses and increase their knowledge. The search box is in the center and is very attention catching for the users. So the first thing a user in this category would do is to enter book names s/he has in mind, if s/he comes with purpose of finding a very specific book. If the user doesn't have a book or article in mind, it is rather unlikely that the user would visit this website, since it is not easy to browse book options and new release on this website.

For users who doesn't read very often or are paperback lovers, the website would be rather confusing, as there's no clear instruction on what each button's functions are, what the users should input to the search box; how the users can get access to the source of the books or articles. The lack of information on the front page would cause this confusion. But after navigating the website for multiple times, the users should be able to use it rather effortlessly.




Efficiency

The website should be pretty efficient for users who have a clear goal of finding a book, but less so for those who want to explore the website and see what they have to offer, since there are too many information listed without a clear structure.

The website is pretty inefficient for those who upload information, as there are many options, less hierarchy between options, making it hard to find things sometimes.




Memorability

The front page is pretty memorable, since old users of the Libgen website are rather unlikely to forget the steps to get access to their desired books or articles. However, old users are unlikely to explore other parts of the front page, since the webpage is rather disorganized and it is hard to tell the hierarchy between different links and dropdowns.

The other type of users, the users who would upload the media files they acquired to the website are more familiar with different buttons, dropdowns, and where each link leads to. It wouldn't be hard to them to pick up where everything is for them to upload information. Yet the website is not ideal for them to navigate, especially with multiple options under each dropdown, and there isn't much color contrast between different buttons to help with memorization.




Satisfaction

The user satisfaction of the website should be moderate, given that the user is happy about finding the materials s/he needs or uploading new content for public browsing. However, satisfaction rate might decrease due to convolusion in operating the front page and the general searching experience.




Few Errors

In general, the website doesn't have much errors, but sometimes the link can lead users to very confusing pages, andthe labeling of many buttons are not really understandable. However, the amount of errors or indistinguishable text through clicking some link would leave users confused and the user experience isn't optimal.

Commentary

A very direct observation from evaluating the accessibility of the Libgen website is that many tags are widely exposed directly in the center of the front page, making the front page very noisy and unstructured. There are also two lists that are separated into top navbar and bottom navbar, which is not really efficient for users.

From Low-fidelity to High-fidelity

The process from Low-fidelity wireframing to High-fidelity prototyping is a process of evaluating what's aesthetic, what's not; It's a process of adding more possibility of interaction to see which elements in a low-fi wireframe is unreasonable. In this process, I decided upon a rather harmonious while not boring color palette to show consistency but also contrast when necessary. I categorized elements so they belong to different groups to create a visual hierarchy. I used rather consistent fonts to add to the hierarchy. Ultimately, I used this layout to emphasize important information to increase efficiency. There are changes from the low-fi to high-fi structure, as I thought the latter more aesthetic and creates more spatial dynamics.

Final Comments

Hope you enjoy using the redesigned Libgen website! In the final responsive redesign, I decided to make the break points for navbar smaller so that tablets could also have a holistic view of all the navbar items. I make this decision based on the fact that many tablets serve the same function as laptops for many people today. Since the functionality are similar, the design of the navbar should resemble one another. In addition, to increase consistency for all the cards, I added "Read More" option to the "Alternative Domains" card. The design of the dropdowns are more modern and that in the prototype.
